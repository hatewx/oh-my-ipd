# IPD-OMO 工作流技能

## IPD 阶段映射

华为 IPD 流程映射到 OMO 执行流程：

```
华为 IPD 阶段          OMO 阶段                 交付物
─────────────────────────────────────────────────────────
概念阶段 (Concept)     Charter 定义              Charter.md
─────────────────────────────────────────────────────────
计划阶段 (Plan)        LLD 设计                  LLD.md
                       测试计划                  Test-Plan.md
─────────────────────────────────────────────────────────
开发阶段 (Develop)     编码实现                  代码 + 单元测试
                       TR1/TR2                   TR-Reports/
─────────────────────────────────────────────────────────
验证阶段 (Qualify)     TR3                       TR-Report-3.md
                       系统测试                  Test-Results/
─────────────────────────────────────────────────────────
发布阶段 (Launch)      Release                   Release-Notes.md
─────────────────────────────────────────────────────────
生命周期 (Lifecycle)   维护迭代                  持续 TR
```

## 阶段 1: Charter 定义 (Concept Phase)

### 目标
明确项目需求基线，定义 MVP 范围，获得立项批准。

### 参与角色
- **主导**: PDU
- **配合**: SE (技术可行性评估)
- **审批**: LPDT

### 检查清单
- [ ] 问题陈述清晰
- [ ] 目标用户明确
- [ ] 需求基线完整 (FR/NFR)
- [ ] 优先级合理 (P0/P1/P2)
- [ ] 验收标准可验证
- [ ] MVP 范围明确
- [ ] 风险已识别

## 阶段 3: 开发与 TR (Develop Phase)

### 目标
按计划完成开发，通过 TR1 和 TR2 质量门禁。

### 参与角色
- **主导**: Developer
- **评审**: PQA, TMM, SE, PDU
- **决策**: LPDT

### TR1 (开发完成评审)
#### 检查清单
- [ ] 所有 P0 功能实现
- [ ] 单元测试通过
- [ ] 代码规范检查通过
- [ ] 架构符合 LLD

#### 参与角色
- PQA: 代码规范检查
- SE: 架构符合性评审
- Developer: 修复问题

### TR2 (集成测试评审)
#### 检查清单
- [ ] 集成测试通过
- [ ] 端到端场景通过
- [ ] 性能基线达标
- [ ] 安全扫描通过

#### 参与角色
- TMM: 集成/E2E 测试
- PQA: 安全/质量门禁
- PDU: 需求符合度验证

### TR Dry Run Loop
```
Developer 实现
    ↓
提交 TR
    ↓
PQA 检查 ──Fail──→ 修复
    ↓ Pass
TMM 测试 ──Fail──→ 修复
    ↓ Pass
SE 评审  ──Fail──→ 修复
    ↓ Pass
PDU 验证 ──Fail──→ 修复
    ↓ Pass
LPDT 决策
    ↓ Pass
进入下一阶段
```

## 阶段 4: 系统验证 (Qualify Phase)

### 目标
完成系统级验证，确保产品达到发布标准。

### 参与角色
- **主导**: TMM
- **配合**: PQA (质量检查), Developer (缺陷修复)
- **审批**: LPDT, PDU

### TR3 (系统验证评审)
#### 检查清单
- [ ] 所有测试用例通过
- [ ] 性能测试达标
- [ ] 安全审计通过
- [ ] 文档完整
- [ ] 无 P0/P1 缺陷

### 发布检查单
```markdown
## Release Checklist

### 功能验证
- [ ] P0 功能 100% 通过
- [ ] P1 功能 100% 通过
- [ ] 用户场景验证通过

### 质量验证
- [ ] 代码覆盖率 > 80%
- [ ] Lint 0 errors
- [ ] 安全扫描通过

### 文档验证
- [ ] 用户文档完整
- [ ] API 文档完整
- [ ] 部署文档完整

### 审批
- [ ] PDU 验收通过
- [ ] SE 架构确认
- [ ] LPDT 发布批准
```

## 阶段 5: 发布 (Launch Phase)

### 目标
完成产品发布，交付给用户。

### 交付物
- Release Notes
- 部署包
- 用户文档
- 运维文档

## 迭代边界定义

### 迭代开始条件
- Charter 已批准
- LLD 已完成
- 上一迭代已关闭

### 迭代完成条件
- 所有计划功能实现
- TR 通过
- 文档已更新

### 迭代终止条件
- 无法克服的技术障碍
- 需求重大变更
- 资源耗尽
- LPDT 决策终止
